Programme Structure

Five Phases. One Complete App.

Each day builds on the last. By Day 5, students have a live app on AWS accessible from real Android phones.

Phase 1
Flutter Foundations
Install Flutter, understand widgets and hot reload, write first Dart code. First Claude Code prompts. First working screen on your machine.
✦ First running app
Phase 2
Full App + Prompting Patterns
All 5 screens. Role-based auth, animated transitions, image picker, confirmation dialogs. Learn the prompting patterns that produce exactly what you need.
✦ Complete app on dummy data
Phase 3
Real Database — PostgreSQL
Design a 3-table schema with ENUM types, foreign keys, indexes. Seed with real data. Connect Flutter directly to PostgreSQL.
✦ App backed by PostgreSQL
Phase 4
API Layer — Node.js + Express
Build a REST API with 8 endpoints. Flutter switches from direct DB to HTTP calls. Database credentials leave the app.
✦ Flutter → API → PostgreSQL
Phase 5
Build, Export & Deploy
Export as APK, IPA, EXE. Deploy the API to AWS EC2, migrate the database to AWS RDS. App goes live.
✦ Live on AWS + Play Store
Field Guide
Pre-Launch Essentials
Google Sheets migration. Dev/Test/Prod environments. APK size management. Security checklist — AI code is not automatically secure.
✦ Production ready

Running Locally

Get It Running in 10 Minutes

Prerequisites: Flutter SDK (latest stable) · Node.js v18+ · PostgreSQL 14+ · Claude Code

1 — Database

psql -U postgres -c \
  "CREATE DATABASE khan_chacha;"

psql -U kc_app -d khan_chacha \
  -f db/schema.sql

psql -U kc_app -d khan_chacha \
  -f db/seed.sql

2 — API Server

cd khan_chacha_api
npm install
cp .env.example .env
# fill in your DB credentials

npm run dev
# → API running on :3000

3 — Flutter App

cd khan_chacha_mrms
flutter pub get

flutter run -d windows
# or -d emulator
# or -d chrome

👉 The API must be running before launching Flutter. Change _baseUrl in lib/services/api_service.dart to match your run target. See the Day 4 handbook for platform-specific URLs.
