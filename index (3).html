Programme Structure

Five Days. One Complete App.

Each day has a dedicated interactive handbook — tabbed navigation, copy-ready Claude Code prompts, architecture diagrams, and real-session Q&A.

Day 1
Flutter Foundations
Install Flutter, understand widgets and hot reload, write first Dart code with Claude as pair-programmer. First working screen on your machine.
✦ First running app
Day 2
Full App + Prompting Patterns
All 5 screens built. Role-based auth, animated transitions, image picker, confirmation dialogs. Learn the prompting patterns that make Claude generate exactly what you need.
✦ Complete app on dummy data
Day 3
Real Database — PostgreSQL
Design a 3-table schema with ENUM types, foreign keys, and indexes. Seed with real data. Connect Flutter directly to PostgreSQL.
✦ App backed by PostgreSQL
Day 4
API Layer — Node.js + Express
Build a Node.js REST API with 8 endpoints. Flutter switches from direct DB to HTTP calls. Database credentials leave the app. Production-grade architecture.
✦ Flutter → API → PostgreSQL
Day 5
Build, Export & Deploy
Export the Flutter app as APK for Android, IPA for iOS, EXE for Windows. Deploy the API to AWS EC2, migrate the database to AWS RDS.
✦ Live on AWS + Play Store
Field Guide
Pre-Launch Essentials
Migrate Google Sheets data to PostgreSQL. Dev/Test/Production environments. APK size management. Full security checklist — because AI-generated code is not automatically secure.
✦ Production ready

The Method

Vibe Coding — But With Discipline

Claude is the co-programmer. Students are the architects. Every decision is explained before it is implemented.

💬

Why Before How

Every concept is explained in plain English before any code is written. Students understand what they are building — not just how to paste a prompt.

📋

Step-by-Step + Capstone

Every day gives two paths: 10–12 progressive prompts that teach each piece, or one complete capstone prompt that builds it all at once.

🔍

Real Errors Documented

Every error encountered in live sessions is documented in the handbook Q&A. No clean fictional examples — real mistakes, real fixes.
